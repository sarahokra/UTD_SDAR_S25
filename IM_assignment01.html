3. If data can be retrieved efficiently and effectively, why data mining is needed?

Data retrieval focuses on efficiently locating data already stored in a database. In contrast, data mining applies statistical methods and artificial intelligence to extract valuable insights from existing data. This process assists businesses and researchers with decision-making, forecasting future trends, and even uncovering new knowledge. For example, in healthcare, medical data mining helps identify disease risk factors, categorize patient groups, and determine effective treatment strategies, ultimately enhancing the quality and efficiency of healthcare services. In social media, analyzing user behavior and sentiment on platforms aids businesses in understanding market trends, product reputation, and customer satisfaction, facilitating the development of effective marketing strategies.

4. Why NoSQL systems emerged in the 2000s? Briefly contrast their features with traditional database systems.

After the 2000s, the demand for emerging applications like social media and big data analytics shifted data storage requirements. Data types expanded beyond traditional structured formats to include semi-structured and unstructured formats like JSON and XML. These changes necessitated large-scale storage and rapid scalability. To accommodate high-concurrency read and write operations, NoSQL systems became suitable alternatives to traditional relational database management systems (RDBMS), which rely on row-oriented storage and strict consistency. NoSQL databases offer better flexibility for diverse data types, making them useful for large-scale social media data and real-time queries.

Key Characteristics of NoSQL Systems:

1. Flexible Data Model: NoSQL databases lack a fixed schema, storing semi-structured or unstructured data (e.g., JSON, XML).

2. High Scalability: Their distributed architecture supports cost-effective horizontal scaling compared to traditional vertical scaling.

3. Eventual Consistency: NoSQL allows temporary data inconsistencies, enhancing availability through eventual consistency.

4. Flexible Query Mechanisms: Many NoSQL databases offer custom APIs or alternative query languages instead of SQL.

5. Optimized for Big Data: NoSQL systems excel in handling large data volumes, such as big data storage and log analysis.

NoSQL Systems vs. Traditional RDBMS:

Feature NoSQL Systems Relational Databases (RDBMS)
Data Model Semi-structured or unstructured data: document-based, key-value, and graph-based models. Structured data stored in tables (rows and columns) with a strict schema.
Query Language Custom APIs or NoSQL query languages. SQL (Structured Query Language).
Scalability Horizontal scaling (Scale-out) fits distributed architectures. Vertical scaling (Scale-up) depends on enhancing single-node performance.
Consistency Eventual consistency favors availability and scalability. Strong consistency (ACID) guarantees strict data integrity.
Use Cases Social media, big data applications, distributed systems. Financial transactions, enterprise applications, and traditional business systems.
Performance Advantages Optimized for high-volume read and write operations. Optimized for structured queries and transactional processing.

5. What are the things current database system cannot do?

1. Automated Semantic Understanding: Databases cannot grasp the semantics of data and can only operate according to established rules.

2. Automated Decision-Making Suggestions: Database systems do not possess advanced analytical abilities and depend on external applications for data analysis.
